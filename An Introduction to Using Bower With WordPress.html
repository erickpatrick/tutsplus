Introdução ao Uso do Bower com WordPress

A maioria dos projetos para web, hoje, começam ou incluem bibliotecas de terceiros. Se você está trabalhando em um tema do WordPress, é bem provável que você esteja usando o HTML5 Boilerplate, Bootstrap ou o Foundation. A maioria desses projetos está disponível no Github e acessível através do Bower.

Nessa série, apresentarei o Bower, falarei sobre como ele pode aprimorar o seu desenvolvimento de temas e plugins para WordPress, além de mostrar como prepará-lo para seu projeto.

O Que é o Bower?

O que, exatamente, é o Bower? De acordo com o site do projeto:

Bower é um gerenciador de pacotes para a web. Oferece uma solução genérica e imparcial para o problema de gerenciamento de pacotes front-end, expondo o modelo de dependência de pacotes através de uma API que pode ser consumida por um processo de implementações mais dogmático.

Essencialmente, o que isso significa é que o Bower fornece a você uma maneira de especificar quais bibliotecas (ou pacotes) que seu projeto precisa para ser desenvolvido. Ele também é capaz de lidar com as dependências dos projetos que você especificar, de forma automática.

O Bower depende do Node.js e do NPM para funcionar, assim, você precisa ter certeza que os tem instalados em sua máquina. Outro requerimento para trabalhar com o Bower é que os repositórios precisam estar no GitHub e ser adicionados ao Catálogo do Bower.

Bibliotecas de Terceiros

Como mencionei antes, há uma grande chance que você esteja usando uma biblioteca de terceiro em seu projeto. Você provavelmente já foi ao GitHub, baixou algum projeto e colocou os arquivos em um local apropriado dentro do seu projeto. Você precisaria repetir essa tarefa para toda e cada biblioteca que seu projeto precisar.

Com o Bower, você pode instalar (ou baixar), facilmente, essas bibliotecas, sem precisar ir ao GitHub. Todas as bibliotecas que você instala com o Bower serão colocadas em um diretório, bower_components, dentro do diretório raiz do seu projeto. Tudo isso pode ser feito pela linha de comando, assim, não há necessidade de ir e vir do navegador o tempo todo.

Administrando Dependências

Outro grande recurso do Bower é que ele administra as dependências para você. Ele busca todos os pacotes que seu projeto precisa e todos os pacotes que esses pacotes precisam também. Cada pacote pode requerer uma versão específica de outro pacote, como, por exemplo, uma versão específica da jQuery.

Então, por exemplo, digamos que seu projeto requeira a jQuery. Você também decidie que precisará usar tanto o Bootstrap quanto o FitVids. O Bower verifica todos os pacotes e todas as versões dos pacotes dos quais eles dependem, avaliando quais versões são requeridas por todos. Se houver algum conflito, ele avisará.

Quem quer ter de lidar com todos esses requerimentos, mesmo?

Projeto Esguio

Como já mencionado, todos os pacotes que você instalar, serão colocados no diretório bower_components. Você pode acessar qualquer coisa dentro desse diretório enquanto estiver em seu ambiente de desenvolvimento, mas, você não precisa adicioná-lo ao repositório do seu projeto.

Digamos que você use o Bootstrap para o layout do seu tema e que queira usar o Sass para compilar suas folhas de estilo em um único arquivo style.css. Você pode usar o pacote bootstrap-sass-official e, simplesmente, importar o arquivo bootstrap.scss para os arquivos .scss já existentes do seu projeto.

Entrarei em mais detalhes em como fazer isso em outro post.
