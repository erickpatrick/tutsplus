Programação Orientada a Objetos no WordPress: Introdução

Uma dos motivos para o pessoal amar (ou odiar, mas isso é assunto para outra série) o WordPress é a necessidade de um baixo nível de conhecimento para começar a usá-lo. 

Isso quer dizer que, para o programador mediano, é bem fácil escolhê-lo e começar a criar projetos o utilizando, especialmente se você for acostumado com desenvolvimento front-end.

Claro, se estiver interessado em extender o WordPress através de plugins, widgets ou mesmo através da construção de uma aplicação web baseada em WordPress, a curva de aprendizado é um pouco mais acentuada. Na verdade, para construir widgets, você, geralmente, tem de lançar mão de programação orientada a objetos se quiser seguir o modelo provido pelo Codex. Além do mais, tem se tornado bem comum ver o paradigma de orientação a objetos sendo usado para plugins famoso, também.

Esse paradigma - embora não seja a solução perfeita como alguns tendem a acreditar - é bem popular por inúmeras razões, mas, antes de falarmos sobre isso, é importante entender o propósito desta série e o que veremos durante as próximas semanas.

"Por Onde Começo?"

Para qualquer pessoa minimamente interessada em aprender uma nova linugagem, ferramenta ou biblioteca, é relativamente fácil começar, especialmente se tiver algum conhecimento em programação.

Mas, aqueles que estão há muito tempo nessa área, costumam esquecer como era passar por momentos que se tentava descobrir o que um código fazia, entender o porque de algo ter sido escrito da forma como o foi, como o autor sabia o que usar e onde, e como descobrir os motivos de certas escolhas para implementação.

Todos passamos por isso um dia, não é? Já olhamos para códigos tentnado descobrir como que ele funcionava e até que chegava um hora que perguntamos "Por Onde Começo?"

And the purpose of this series is to answer that question.

Object-Oriented Programming For Beginners

Specifically, this series is going to focus solely on those who are wanting to learn to build more complex solutions with WordPress. You may be someone who is comfortable with building themes and/or simple plugins, but are looking to advance your skill set by learning object-oriented programming.

The challenge, of course, is learning where to start. Sure, there's a lot of material that's available on the web and in books in terms of the basic concepts of object-oriented programming, but I believe that it's often easiest to learn a new paradigm within the context of an actual foundation or framework.

So over the next few articles, we're going to start from the ground up by looking at object-oriented programming within the context of WordPress.

We'll be covering topics such as:

  • classes
  • attributes
  • constructors
  • functions
  • scope
  • interfaces
  • abstract classes
  • abstract functions
  • ...and much more.

By the end of the series, you be well-equipped to at least have a working understanding, and a solid reference that you can look back to throughout your time with WordPress.

What's Next?

Ultimately, we want to work on not only understanding how object-oriented programming works, but also work on putting it to practical use by building a working plugin. So as we cover the basics of object-oriented programming within the context of WordPress, we're going to work on building a plugin to demonstrate the features of the paradigm.

With that said, we're ready to get started.
