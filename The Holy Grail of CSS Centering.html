O Santo Graal da Centralização com CSS

Chris Coyier publicou recentemente a fluxo de decisão em seu site, CSS Tricks, para demonstrar a quantidade de métodos para centralizar coisas, verticalmente e horizontalmente, usando CSS. Como soluções individuais, elas tem lá seus problemas, contudo, combinadas, são uma ferramenta poderosa para centralizar as coisas, que funciona entre navegadores, de forma consistente, a partir do IE8.

É assim que se faz.

Blocos Iniciais

Se você está criando um site responsivo, provavelmente estará dimensionando seus elementos em percentuais, então, criemos um elemento .container qualquer:

<div class="container">
  ...
</div>

.container {
  width: 70%;
  height: 70%;
  margin: 40px auto;
  background: red;
}

Para garantir que nossos elementos se estendem apropriadamente, atribuamos html, body { width: 100%; height: 100%; }.

Virando a Mesa

Agora, dentro de .container, centralizaremos de forma vertical com o conhecido padrão das tabelas. Isso faz com que elementos block comuns se comportem como células de tabelas, permitindo-nos centralizar verticalmente:

<div class="container">
  <div class="outer">
    <div class="inner">
      ...
    </div>
  </div>
</div>

.outer {
  display: table;
  width: 100%;
  height: 100%;
}
.inner {
  display: table-cell;
  vertical-align: middle;
  text-align: center;
}

Por fim, adicionaremos um elemento centralizado horizontalmente:

<div class="container">
  <div class="outer">
    <div class="inner">
      <div class="centered">
        ...
      </div>
    </div>
  </div>
</div>

.centered {
  position: relative;
  display: inline-block;

  width: 50%;
  padding: 1em;
  background: orange;
  color: white;
}

Inside of that, you can place anything your heart desires, including dynamic height text blocks, or absolutely positioned elements.

Alterations

To change the horizontal centering of the element, just modify .inner's text-align property. To change the vertical centering, modify .inner's vertical-align property.

In Conclusion

This example may seem like a lot of markup, but the working pieces are just three divs: .outer, .inner, and.centered. And the styling for these is consistent so you can use this in your boilerplate CSS across projects and never worry about it again.

It's one more element larger than most centering techniques and completely bulletproof. My hope is that this becomes the standard centering approach and we can finally stop worrying about centering in CSS.
